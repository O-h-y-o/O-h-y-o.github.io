int num 으로 동일하게 했을때 이게 왜 자꾸 오답이지 하면서 여러가지 식을 계속 바꿔봤는데도 안되길래 머리가 아팠는데,, n \* 3 이 엄청 커질 수 있어서 int의 범위를 벗어나기 때문이었다아..

```cpp
int solution(int num) {
    int answer = 0;
    long long n = num;

    while (n != 1) {
        n = (n & 1) == 0 ? n / 2 : n * 3 + 1;
        answer++;

        if(answer >= 500) return -1;
    }

    return answer;
}
```

## C++ 기초 복습 시스템

TextRPG를 코딩하다가 문득 내 코딩 속도가 너무 느리다는 걸 느꼈다. 왜 이렇게 느릴까 생각해보니, C++ 기초를 정확하게 숙지하지 못하고 있어서였다. 머리속으로는 설계가 그려져도, 코딩할 때마다 "이거 어떻게 했더라?", "저건 어떻게 하는 거였지?" 하면서 계속 찾아보는게 맞는건가 싶었다. 계속 머리에서 지워지는 느낌..?

그래서 개념을 직접 내 말로 설명해보고, 코드도 직접 짜본 다음 답을 확인할 수 있는 복습 시스템을 만들어봤다.

- 개념 질문에 내가 직접 설명을 적고 답안을 확인하는 방식
- 코드도 직접 작성해보고 예시 코드와 비교

변수부터 시작해서 포인터/메모리, 클래스/캡슐화, 상속/다형성, 오버로딩/템플릿, STL, 컴파일/스코프, 예외 처리, 설계 원칙까지 주제별로 나눠서 정리했고, RAII나 Rule of Three처럼 알아야 하는데 놓치고 있던 개념들도 하나씩 채워나가는 중이다.
앞으로도 새로운 개념들이 있으면 채워나갈거다.

어떤 것이든 기초가 가장 중요한 법이니까, 누군가 OO 에 대해서 설명해보세요 라고 했을때 줄줄 나오는 것 까진 아니더라도, 정확하게 설명할 수 있을 만큼은 되어야 한다. 시스템도 만들었고, 정리도 했으니 내일부턴 에이타니 + 코트카타 → c++ 개념 무한복습이다. 시간이 남으면 textrpg를 조금 더 개발할..?수도..
